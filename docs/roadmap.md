## Phase 1: MVP (6~8주)

목표: 공공 API + 사용자 URL 추가 기능으로 서비스 오픈, 첫 유료 고객 확보

```
Week 1-2: 프로젝트 셋업 + DB + 인증
  ├── Next.js + FastAPI 프로젝트 구조
  ├── PostgreSQL 스키마 생성 (Alembic 마이그레이션)
  ├── JWT 인증 (회원가입/로그인/토큰갱신)
  ├── 테넌트 생성 + 사용자 관리
  └── bid-collectors 패키지 연동 (나라장터/K-Startup/기업마당)

Week 3-4: 수집 파이프라인 + AI 스크래퍼
  ├── Celery + Redis 셋업
  ├── bid-collectors → bid_notices 수집 파이프라인
  ├── Celery Beat 정기 수집 (하루 2회)
  ├── AI 스크래퍼 생성 모듈 (URL → Claude API → scraper_config)
  ├── generic_scraper 엔진 연동 (scraper_registry → scraped_notices)
  ├── URL 추가 → AI 분석 → 테스트 수집 → 미리보기 → 구독 플로우
  └── 스크래퍼 정기 수집 (Celery Beat, subscriber_count > 0만)

Week 5-6: 조회 + 태그 + 키워드 + 대시보드
  ├── 공고 목록 API (bid_notices UNION scraped_notices, 테넌트 키워드 FTS)
  ├── 공고 리스트/상세 프론트엔드
  ├── 태그 시스템 (검토요청/입찰대상/제외)
  ├── 키워드 관리 UI
  ├── 출처 관리 UI (공공 API 상태 + 구독 목록 + URL 추가)
  ├── 대시보드 (신규 공고 수, 키워드 수, 마감 임박)
  └── 이메일 알림 (신규 공고 + 마감 임박)

Week 7-8: 결제 + 랜딩 + 배포
  ├── 토스페이먼츠 정기결제 연동
  ├── 요금제별 기능 제한 (키워드 수, URL 수)
  ├── 랜딩 페이지 (서비스 소개 + 요금제)
  ├── Docker Compose 배포 (EC2)
  ├── 도메인 + SSL + Nginx
  └── 베타 테스트 + 버그 수정
```

**병렬 작업: bid-collectors 패키지**
- MVP 기간 동안 별도 프로젝트로 병렬 개발
- 1단계 API: 나라장터 입찰공고 + 기업마당 + 보조금24
- BaseCollector 인터페이스 확정 후 독립 진행

## Phase 2: 안정화 + UX 개선 (3~4주)

```
Week 9-10: AI 스크래퍼 고도화
  ├── AI 분석 성공률 개선 (실패 사례 학습, 프롬프트 개선)
  ├── 스크래퍼 모니터링 (사이트 구조 변경 감지, 자동 알림)
  ├── 실패 시 AI 자동 재분석 시도
  └── 스크래퍼 깨짐 → AI 자동 복구 파이프라인

Week 11-12: UX + 성능
  ├── URL 추가 UX 개선 (분석 진행 상태, 미리보기 상세화)
  ├── 수집 이력/통계 페이지
  ├── 성능 최적화 (공고 10만건+ 대응)
  └── 고객 피드백 반영
```

**병렬 작업: bid-collectors 2단계**
- LH/KEPCO/도로공사/수자원공사 자체 전자조달 API
- 방위사업청 군수품조달 API

---

## Phase 3: 프리미엄 — AI 매칭 (3~4주)

```
Week 13-14: 프로필 + 매칭
  ├── 회사 프로필 입력 UI
  ├── AI 매칭 배치 (Celery — 매일 아침)
  ├── "내게 맞는 공고" 페이지
  ├── 매칭 점수 + 이유 표시
  └── 카카오 알림톡 연동

Week 15-16: 첨부파일 분석 + 고도화
  ├── 첨부파일 다운로드 + PDF/HWPX/DOCX 파싱
  ├── AI 첨부 분석 (자격요건 체크)
  ├── AI 채팅 (공고 Q&A)
  └── 프리미엄 요금제 오픈
```

**병렬 작업: bid-collectors 3단계**
- K-Startup + 중소벤처24 (창업/중소기업 특화)
- 나라장터 낙찰/계약 API (입찰 전후 프로세스)

## 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| AI 스크래퍼 생성 성공률 낮음 | **높** | 서비스 핵심 기능. 프롬프트 지속 개선, 실패 사례 DB 누적, 수동 대응 팀 |
| 사이트 구조 변경으로 스크래퍼 깨짐 | 중 | 수집 실패 모니터링 + AI 자동 재분석 + 고객 알림 |
| 나라장터 API 정책 변경 | 높 | API 호출량 모니터링, 대안 수집 방법 준비 |
| 경쟁사 가격 경쟁 | 중 | "내 URL 직접 추가" + AI 매칭 차별점 강화 |
| 공고 데이터 저작권 이슈 | 낮 | 공공데이터 — 자유 이용 가능 (공공데이터포털 약관) |
| 스크래핑 차단 (IP 밴) | 중 | 요청 간격 조절, User-Agent 로테이션, 프록시 |
| URL 수집 비용 증가 | 낮 | scraper_registry로 URL당 1회만 수집, 구독 테넌트 수와 무관 |
